# Domain 7 — Benefits of the GitHub Community

## Open source on GitHub

| Concept | Meaning |
|---|---|
| Open source | Code with a license allowing use, study, modification, sharing |
| **License** | Defines what others may do (MIT, Apache 2.0, GPL, etc.) — without one, default copyright applies |
| Fork & PR | The standard way outsiders contribute |
| Star | Bookmark/show appreciation for a repo |
| Watch | Subscribe to a repo's notifications |

> **No license = all rights reserved.** A repo without a LICENSE is *not* free to reuse, even if public.

## Community health files

GitHub recognizes files (in the repo or a shared `.github` repo) that make projects welcoming and well-run:

| File | Purpose |
|---|---|
| `README.md` | What the project is and how to start |
| `CONTRIBUTING.md` | How to contribute |
| `CODE_OF_CONDUCT.md` | Expected behavior |
| `LICENSE` | Legal terms |
| `SECURITY.md` | How to report vulnerabilities |
| Issue/PR templates | Standardize submissions |

A repo's **Community Standards** page shows which of these are present.

## Your GitHub profile

| Feature | Detail |
|---|---|
| **Profile README** | A repo named the same as your username; its README renders on your profile |
| Pinned repositories | Highlight up to 6 repos |
| **Achievements** | Badges earned through activity |
| Contribution graph | Visual history of your activity |
| Followers / Following | Social graph |

## GitHub Sponsors

Lets the community **fund developers and projects** financially. Maintainers add a `FUNDING.yml` to show a Sponsor button.

## Learning & support resources

| Resource | What it offers |
|---|---|
| **GitHub Docs** | Official product documentation |
| **GitHub Skills** | Interactive, hands-on courses |
| **GitHub Community** | Discussions/forum for help |
| GitHub Blog / Changelog | Product news and updates |
| GitHub Education | Free tools for students/teachers (Student Developer Pack) |

## InnerSource

Applying **open-source collaboration practices inside a company** — using internal/private repos so teams across an organization can discover, reuse, and contribute to each other's code. **Internal** repository visibility supports this.

## Exam traps

1. **No license = not reusable** (default copyright), even on public repos.
2. **Star vs Watch** — star = bookmark; watch = subscribe to notifications.
3. **Profile README** comes from a repo named exactly your username.
4. **InnerSource** = open-source practices applied internally (often via internal repos).
5. **Sponsors** is for funding; **Achievements** are activity badges.
