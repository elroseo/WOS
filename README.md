# wos

My **work operating system**: the Obsidian vault and Copilot CLI setup I use to run my day as a CRE at GitHub.

It is version controlled with `obsidian-git` so notes, config, and tooling stay backed up and portable.

## What's inside

- **Environment/** - my Copilot CLI setup: custom agents, skills, and dev environment notes
- **Templates/** - reusable Obsidian templates (1-on-1s, meeting notes, incidents, hub notes)
- **Cheatsheets/** - quick references (Kusto/KQL, Splunk, GHES, GHAS, Git, SSH, SSL/TLS, and more)
- **Notes/** - product deep dives (GHEC, GHES, GHAS, Actions) and meeting notes
- **Training & Development/** - certification paths and study notes (GitHub Foundations, etc.)
- **People/** - lightweight profiles and 1-on-1 context
- **Daily/** - daily plans generated by the `today` skill
- **Voice/** - writing style, tone guide, and goals
- **.obsidian/** - vault config and plugins (Tasks, Templater, Git, Local REST API)

## How I use it

- **Morning**: run the `today` skill for a daily briefing (tasks, meetings, Slack)
- **Before meetings**: run the `meeting` skill to prep context and notes
- **End of day**: run the `eod` skill for a summary and shareable update
- **Friday**: run the `week` skill for a weekly review

Skills read and write to this vault through the Obsidian Local REST API plugin.
