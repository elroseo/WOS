# Writing in GitHub Voice - Notes

Source: Stefan Signell's presentation on writing in GitHub voice and tone (internal training)

## Key Takeaways

### Why we have voice guidelines
1. **Make everyone content creators** - guidelines help all teams write consistently
2. **Speak to people like people** - avoid jargon, be accessible, acknowledge the human aspect
3. **Create one brand experience** - from the audience's perspective, it's all just "GitHub"

### Brand attributes applied to writing
| Attribute | Application |
|-----------|-------------|
| Authentic | Talk like a person, not a robot |
| Nerdy | Tell technical stories with enthusiasm |
| Empathetic | Consider your audience's context |
| Imaginative | Show optimism and soul |
| Confident | Be a resource, speak with authority |

### Writing for developers
- Dial up: technical details, authenticity
- Dial down: marketing language, jargon
- Be precise, practical, and useful
- Don't explain common acronyms (HTML, CI/CD)
- Add personality with relevant references
- Every sentence should add value

### Writing for enterprises
- Dial up: empathy, confidence
- Dial down: overly technical details
- Speak to challenges software teams face
- Be a source of inspiration and optimism
- Enterprise buzzwords are like salt: sprinkle, don't pour
- Don't be playful about serious topics (security, compliance)
- Ensure content is scannable for busy readers

## Pro Tips (from Stefan)

### 1. Less is more
> "Perfection is not achieved when there's nothing more to add, but when there's nothing left to take away." - Antoine de Saint-Exupéry

- Don't repeat yourself (DRY applies to writing too)
- Avoid being verbose. If you can cut 25%, do it.
- Assume the reader is busy and has 2 minutes
- Strip down to core components without losing value

### 2. Emphasis is more important than completeness
- Every diagram/document should answer a specific question
- Don't go into the periphery. Link to details instead.
- The "five second rule": flash a diagram for 5 seconds. Can they get the point?
- Everything should serve a purpose. No filler.

### 3. Create a cohesive story
- Follow a linear, coherent narrative
- Don't flip between topics. Group related ideas.
- Humans think linearly. Help them follow along.

### 4. Style matters
- First impressions determine if someone reads further
- Clean, appealing formatting scales the reach of your content
- Use visuals purposefully (not as decoration)
- Simplified language aligned to the audience
- Fonts, spacing, and structure all contribute to readability

### 5. Writing as a learning tool
- If you can't write it clearly, you haven't fully understood it
- Vague words are a signal you need to go deeper
- Better reading makes better writing. Seek out good examples.

## Applying it daily
- Review PRs with these guidelines in mind
- Even 25% improvement in clarity is worth the effort
- Content scales. Hundreds or thousands may read what you write.
- Spend extra time upfront. It compounds over time.
