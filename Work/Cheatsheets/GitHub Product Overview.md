# GitHub Product Overview

## Quick map

**GitHub has three layers:** base plans, enterprise deployment choices, and add-on or companion products.
**Start every customer conversation with deployment, identity, and compliance needs.** That usually narrows the right product set fast.

## Plans

| Plan | What it is | Key differentiator | Best for |
| --- | --- | --- | --- |
| **GitHub Free** | Core GitHub for individuals and small teams. Includes unlimited public repos, unlimited private repos with a lighter feature set, Actions minutes, Packages storage, and basic collaboration. | **Lowest friction entry point.** Great for open source and early team adoption. | **Individuals, OSS maintainers, small teams** |
| **GitHub Team** | Paid organization plan with stronger collaboration, review, and admin features for private work. Also unlocks purchase of advanced security products. | **Full private repo collaboration without enterprise governance overhead.** | **SMB teams, startups, department-level engineering groups** |
| **GitHub Enterprise** | Enterprise plan with centralized billing, policy, identity, security, and deployment controls. Available as **GHEC** or **GHES**. | **Top-down administration at scale.** | **Large companies, regulated orgs, multi-org enterprises** |

## Deployment options

| Option | What it is | Key differentiator | Best for |
| --- | --- | --- | --- |
| **GitHub Enterprise Cloud (GHEC)** | GitHub-hosted enterprise deployment on GitHub.com, or on **GHE.com** when using data residency. | **Fastest access to new features, less infrastructure to manage.** | **Most enterprises, especially cloud-first teams** |
| **GitHub Enterprise Server (GHES)** | Self-hosted GitHub appliance deployed in a private cloud, datacenter, or controlled infrastructure. | **Maximum infrastructure control.** | **Highly regulated, isolated, or customer-managed environments** |

## GitHub Copilot

### Copilot plans

| Plan | What it is | Key differentiator | Best for |
| --- | --- | --- | --- |
| **Copilot Individual** | AI coding assistant for one developer. Delivers inline completions, chat, and workflow acceleration in supported IDEs and the CLI. | **Personal productivity purchase.** | **Individual developers** |
| **Copilot Business** | Organization-managed Copilot with policy controls, seat management, and business admin workflows. | **Managed rollout at org level.** | **Teams standardizing AI-assisted development** |
| **Copilot Enterprise** | Enterprise-managed Copilot with broader policy and enterprise context options. Often the right fit when customers want governance plus deeper internal relevance. | **Enterprise policy and scale.** | **Large enterprises with security, compliance, and admin requirements** |

## Core platform and companion products

| Product | Description | Key differentiator | Who it is for |
| --- | --- | --- | --- |
| **GitHub Actions** | Native CI/CD and automation platform built into GitHub. It runs workflows for build, test, deploy, release, and repo automation. | **Lives where the code and events already are.** | **Dev, platform, and release teams** |
| **GitHub Packages** | Package hosting for containers and common package ecosystems. Works with GitHub permissions and workflows. | **Tight repo and Actions integration.** | **Teams publishing internal or public packages** |
| **GitHub Pages** | Static site hosting for docs, project sites, and lightweight web pages. Publishes directly from a repo or Actions workflow. | **Simple repo-backed publishing.** | **Docs teams, OSS projects, developer portals** |
| **GitHub Codespaces** | Cloud development environments with repo-based configuration and fast onboarding. Developers get repeatable dev environments in the browser or VS Code. | **Consistent dev environments without local setup drift.** | **Platform teams, onboarding-heavy orgs, secure dev setups** |
| **GitHub Projects** | Work planning and tracking with tables, boards, roadmaps, and automation. It connects issues, PRs, and custom fields. | **Planning that stays close to engineering execution.** | **Engineering managers, product teams, program leads** |
| **GitHub Discussions** | Community forum inside a repository or organization. It separates longer-form conversations from issues and PRs. | **Structured community conversation.** | **OSS maintainers, internal communities, support-like workflows** |
| **GitHub Sponsors** | Funding system for open source maintainers and projects. Sponsors can support individuals or organizations. | **Native monetization for open source work.** | **OSS maintainers, foundations, sponsor programs** |
| **GitHub Mobile** | Mobile app for triage, notifications, code review, and basic collaboration on the go. It is not a full desktop replacement. | **Fast response from anywhere.** | **Developers, managers, on-call reviewers** |
| **GitHub Desktop** | Desktop Git client with a friendly UI for common Git and GitHub workflows. Useful for commits, branches, PR prep, and history review. | **Low-friction Git UX.** | **Developers who prefer GUI Git workflows** |
| **GitHub CLI** | Command line interface for GitHub workflows like PRs, issues, releases, Actions, and automation. It also hosts Copilot in the terminal. | **Scriptable GitHub operations.** | **Power users, platform teams, automation-heavy developers** |

## How to position the stack quickly

- **Free** proves value.
- **Team** professionalizes collaboration.
- **Enterprise** adds central control, identity, compliance, and scale.
- **Copilot, security, and developer platform products** deepen value after the core platform lands.

> [!note]
> **Public docs change often.** Re-check pricing, included minutes, storage, and current Copilot packaging before a customer-facing conversation.

> [!tip]
> **CRE tips**
> - Ask **cloud or self-hosted** first.
> - Ask **personal accounts or managed identities** second.
> - Ask **security add-ons, Actions scale, and migration source** third.
> - If the customer sounds unique, they usually are not. Their pattern is often one of these: startup on Team, enterprise on GHEC, or regulated hybrid with GHES plus cloud services.
