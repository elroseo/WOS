## Goal

Master the full anatomy of a GitHub repository, not just the basics but the settings, structure, and behaviors that matter for enterprise customers.

## Key concepts

- Repository types: public, private, internal
- Repository settings: branches, tags, topics, visibility
- Repository templates and forks
- Archive and transfer operations
- Repository rulesets (replacing branch protection rules)
- Default branch configuration
- Large file storage (LFS)

## Learning tasks

- [ ] Create a test repo and explore every settings tab
- [ ] Configure a branch ruleset end to end
- [ ] Experiment with a fork and understand the relationship to upstream
- [ ] Set up Git LFS on a test repository
