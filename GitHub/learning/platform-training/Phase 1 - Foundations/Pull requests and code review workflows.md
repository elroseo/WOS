## Goal

Understand the full PR lifecycle, including enterprise-relevant features like required reviewers, CODEOWNERS, and merge strategies.

## Key concepts

- PR lifecycle: draft, open, review, merge, close
- CODEOWNERS file and auto-assignment
- Review types: comment, approve, request changes
- Merge strategies: merge commit, squash, rebase
- Auto-merge and merge queues
- PR templates
- Required status checks

## Learning tasks

- [ ] Create a PR and go through a full review cycle
- [ ] Set up a CODEOWNERS file and verify auto-assignment
- [ ] Configure required reviewers and status checks on a branch
- [ ] Test all three merge strategies and understand the commit history differences
