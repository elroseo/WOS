## Goal

Get hands-on with the tools and workflows you'll use day-to-day as a CRE.

## Key concepts

- Internal GitHub tooling for CREs (stafftools, etc.)
- How to investigate customer issues: tracing through orgs, users, repos, apps
- Common customer issue patterns and diagnostic approaches
- Escalation paths: CRE to Engineering
- Incident response and customer communication
- How to read and interpret GitHub's status page and incident history

## Learning tasks

- [ ] Shadow a senior CRE on a live customer case
- [ ] Walk through the last 3 GitHub incidents and understand the root cause for each
- [ ] Practice diagnosing a permission issue end to end (org role, repo role, SAML, team membership)
- [ ] Run through the escalation process for a simulated P1
