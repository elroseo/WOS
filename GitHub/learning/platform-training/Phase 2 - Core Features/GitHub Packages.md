## Goal

Understand GitHub Packages as an artifact registry. Customers frequently use it alongside Actions for CI/CD pipelines.

## Key concepts

- Supported package types: npm, Maven, RubyGems, Docker/OCI, NuGet, Gradle
- Packages vs Container registry (ghcr.io)
- Visibility and access control for packages
- Linking packages to repos
- Retention and deletion policies
- Storage billing for packages

## Learning tasks

- [ ] Publish a Docker image to ghcr.io via a GitHub Actions workflow
- [ ] Set package visibility and verify access controls
- [ ] Configure a retention policy on a package
- [ ] Understand how packages storage counts toward billing
