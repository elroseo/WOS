## Goal

Build a solid working understanding of GitHub Actions, one of the most common sources of enterprise customer questions and issues.

## Key concepts

- Workflow anatomy: triggers, jobs, steps, runners
- Events that trigger workflows (push, PR, schedule, workflow_dispatch, etc.)
- Runners: GitHub-hosted vs self-hosted
- Actions: uses, marketplace, composite actions
- Secrets and variables
- Environments and deployment protection rules
- Concurrency and cancellation
- Caching dependencies
- Workflow permissions and GITHUB_TOKEN

## Learning tasks

- [ ] Write a workflow from scratch that lints, tests, and deploys
- [ ] Use the matrix strategy to test across multiple environments
- [ ] Set up a self-hosted runner and run a workflow on it
- [ ] Configure an environment with required reviewers before deployment
- [ ] Debug a failing workflow using workflow run logs and re-run with debug logging
